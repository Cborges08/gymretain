## Spacing Scale

Declared values (8-point scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, between form fields |
| sm | 8px | Compact element spacing, form label to input |
| md | 16px | Default element spacing, card padding, button padding |
| lg | 24px | Section padding, page header spacing |
| xl | 32px | Layout gaps, sidebar width (64 * 2) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Implementation:** Use Tailwind spacing utilities (px-4 = 16px, py-2 = 8px, gap-4 = 16px, etc.)

## Copywriting Contract

### Primary Actions

| Element | Copy | Context |
|---------|------|---------|
| Create member button | "+ Novo Membro" | Page header, links to `/dashboard/members/new` |
| Save button | "Salvar" | Form submit, both create and edit forms |
| Cancel button | "Cancelar" | Form cancel, navigates back to `/dashboard/members` |
| Print QR button | "Imprimir QR Code" | QR code page action |
| Edit button | "Editar" | Member profile page, links to edit route |
| Deactivate button | "Desativar" | Member profile page, toggles status to inactive |
| Reactivate button | "Reativar" | Member profile page (when status = inactive), toggles status to active |

### Form Labels

| Field | Label | Required |
|-------|-------|----------|
| Name | "Nome" | Yes (required) |
| Email | "Email" | Yes (required) |
| CPF | "CPF" | Yes (required) |
| Phone | "Telefone (opcional)" | No |

### Validation & Error Messages

| Error Type | Copy | Example |
|-----------|------|---------|
| Required field | "{field} é obrigatório" | "Email é obrigatório" |
| Duplicate email | "Este email já está em uso nesta academia" | (shown inline below email field) |
| Duplicate CPF | "Este CPF já está em uso nesta academia" | (shown inline below CPF field) |
| Invalid email format | "Email inválido" | (shown inline below email field) |
| Invalid CPF format | "CPF deve ter 11 dígitos" | (shown inline below CPF field) |

**Error display:** Inline below affected input field, red text (`text-red-500` in Tailwind), same size as label (12px)

### Empty State

**Heading:** "Nenhum membro cadastrado"
**Body:** "Comece criando o primeiro membro da academia. Clique em "+ Novo Membro" para começar."
**Location:** Member list page when `members.length === 0` and `status = 'active'`

### Destructive Actions

| Action | Confirmation | Button Label | Behavior |
|--------|--------------|--------------|----------|
| Deactivate member | "Tem certeza que deseja desativar este membro? Ele não aparecerá na lista de ativos." | "Desativar" | Sets `members.status = 'inactive'`. User remains in history. |
| Reactivate member | None (toggle only, no confirmation) | "Reativar" | Sets `members.status = 'active'`. |

**Note:** Deactivate shows confirmation dialog (toast or modal). Reactivate does not require confirmation.

### Table Column Headers

| Column | Header | Type |
|--------|--------|------|
| Name | "Nome" | sortable (by name) |
| Email | "Email" | text (not sortable) |
| Last Check-in | "Último check-in" | text, displays days or "—" |
| Status | "Status" | badge with color |

### List Row States

- **Normal row:** Hover state with light gray background (`hover:bg-gray-100`)
- **Clickable:** Click on row (or name specifically) navigates to member profile `/dashboard/members/{id}`

## Component Specifications

### Member List Page (`/dashboard/members`)

**Layout:**
- Page header with "Membros" title (24px semibold) and "+ Novo Membro" button (emerald-600)
- Toolbar with optional filter toggle ("Mostrar inativos") — Claude's discretion (Phase 3)
- Table with 4 columns: Nome, Email, Último check-in, Status
- Empty state card (centered) if no active members exist
- Table padding: 16px (md)

**Table styling:**
- Header row: `bg-gray-50`, `border-b border-gray-200`, text `gray-900` 12px semibold
- Data rows: white background, `border-b border-gray-200`, text `gray-900` 14px normal
- Hover rows: `bg-gray-50`
- Row height: ~48px (12px label + 28px padding)

### Member Create/Edit Form (`/dashboard/members/new` and `/dashboard/members/{id}/edit`)

**Layout:**
- Page header with "Novo Membro" (create) or "Editar Membro" (edit), 24px semibold
- Back/breadcrumb: "Membros > Novo Membro" (optional, Claude's discretion)
- Form card: white background, `border border-gray-200`, `rounded-lg`, padding 24px
- Form width: max-width 600px, centered horizontally
- Field spacing: gap-y-4 (16px between fields)

**Form fields:**
- Nome (required): text input, full width
- Email (required): text input, full width
- CPF (required): text input, full width, optional input mask (Claude's discretion)
- Telefone (optional): text input, full width

**Input styling:**
- Height: 40px (Tailwind `h-10`)
- Padding: 8px 12px (sm + xs)
- Border: `border border-gray-300`
- Border radius: `rounded-md`
- Focus: `focus:ring-2 focus:ring-emerald-600 focus:border-transparent`
- Font: 14px normal
- Placeholder color: `text-gray-400`
- Disabled: `bg-gray-50 text-gray-500 cursor-not-allowed`

**Form buttons:**
- "Salvar" button: emerald-600 background, white text, 40px height, full-width or inline with Cancel
- "Cancelar" button: gray background (`bg-gray-200`), gray-900 text, 40px height, inline next to Salvar
- Button spacing: gap-3 (12px) between buttons
- Button padding: px-4 py-2 (16px x 8px)

### Member Profile Page (`/dashboard/members/{id}`)

**Layout:**
- Page header: Member name (24px semibold)
- Back link: "< Membros" (optional)
- Profile card: white background, `border border-gray-200`, `rounded-lg`, padding 24px
- Card sections: separated by `border-t border-gray-200`, padding-y 16px

**Profile sections:**
1. **Basic Info** (always visible)
   - Campo: "Nome" → display value (not editable)
   - Campo: "Email" → display value
   - Campo: "CPF" → display masked value (format: `***.***.***-XX`) unless admin is on edit page
   - Campo: "Telefone" → display value or "—" if empty
   - Campo: "Data de cadastro" → display ISO date (e.g., "2026-03-27") or formatted "27 de março de 2026"
   - Campo: "Último check-in" → display date and days ago (e.g., "2026-03-25 (2 dias atrás)") or "Nunca" if null

2. **Status Section**
   - Status badge: colored according to current status (green/amber/gray)
   - "Editar" button: links to `/dashboard/members/{id}/edit`
   - Deactivate/Reactivate toggle button:
     - If `status = 'active'`: show "Desativar" button, red/destructive style
     - If `status = 'inactive'`: show "Reativar" button, emerald-600 style

3. **Check-in History Section** (placeholder in Phase 3)
   - Heading: "Histórico de check-ins"
   - Placeholder text: "Histórico disponível na fase 5."
   - Or: Empty state if implemented in Phase 3 (but Phase 5 owns this)

### Gym QR Code Page (`/dashboard/qr-code`)

**Layout:**
- Page header: "QR Code da Academia" (24px semibold)
- Instruction text: "Seus alunos escaneiam este código para fazer check-in"
- QR code container: centered, white background card, `border border-gray-200`, `rounded-lg`, padding 32px
- QR code size: 300x300px (or 256x256px based on fitting)
- "Imprimir QR Code" button: emerald-600, positioned below QR code

**QR code generation:**

## Form Validation Contract

### Real-time Validation (Client-side)

- **Email:** Required, valid email format
- **CPF:** Required, format `XXX.XXX.XXX-XX` (11 digits) or plain 11 digits. **Checksum validation deferred to Phase 9.**
- **Name:** Required, non-empty
- **Phone:** Optional, no format required (accept as-is)

### Server-side Validation (Server Action)

- **Email:** Required, unique within org (`UNIQUE(org_id, email)`)
- **CPF:** Required, unique within org (`UNIQUE(org_id, cpf)`)
- **Name:** Required, 1-200 characters
- **Phone:** Optional, max 20 characters
- **Org ID:** Required (from JWT claims, never trust client)

**On duplicate email/CPF:**
- Clear error message displayed inline: "Este {field} já está em uso nesta academia"
- Form is NOT submitted; user can correct and retry

## Accessibility & Interactions

### Keyboard Navigation
- Tab order: inputs flow top-to-bottom in form
- Form label `<label>` associates with `<input id>`
- Buttons are tabbable and respond to Enter/Space

### ARIA Labels
- Form inputs have `aria-label` or `aria-labelledby` for screen readers
- Error messages associated via `aria-describedby`
- Status badges should announce status (e.g., "Status: Ativo" in screen reader)

### Focus States
- All interactive elements show visible focus ring (`:focus-visible` or `:focus`)
- Emerald-600 focus ring (2px) on form inputs
- Underline or color change on links

### Color Contrast
- All text must meet WCAG AA standard (4.5:1 for body text)
- Status badge text: white or dark color on colored background for contrast
- Gray-600 secondary text: acceptable on white background (5.2:1)

---

## Data Masking & Privacy

**CPF Display:**
- In lists and profile view: Masked format `***.***.***-XX` (show only last 2 digits)
- In edit form: Full CPF visible (admin needs to see/correct)
- In server logs: Consider masking in future Phase 9

**Example:**
- Stored: `123.456.789-10`
- Displayed: `***.***.***.89-10` or `***.***.***-89` (show last 2)

## Server Action Pattern

**All member mutations (create, edit, deactivate/reactivate) use Server Actions:**

```typescript
// Example signature (pseudo-code)
async function createMemberAction(formData: FormData) {
  // Validate form data
  // Extract org_id from JWT (auth.jwt())
  // Insert into members table
  // Handle errors (duplicate email/CPF → return error message)
  // Redirect to member list on success
}
```

**Error handling:**
- Validation errors returned to form as inline messages (pt-BR)
- Server errors logged; user sees friendly message
- No exposing internal DB errors to UI
